Persistent 500 Error on Checkout API for Enterprise Accounts

emergency_home Critical pending In Progress payments Billing Core

Description

We've received multiple reports from Tier 1 clients (specifically Acme Corp and Globex) regarding a persistent internal server error when attempting to finalize checkout for bulk orders exceeding 5,000 units. Initial logs suggest a timeout in the /api/v2/transactions/process endpoint.

  • Error manifests after 30 seconds of pending state.
  • Affects only Enterprise tier accounts.
  • Temporary workaround: Splitting orders into smaller batches.
